# 1ain Project Overview

## The Concept
A personal AI operating system where one master agent (1ain) orchestrates all
AI capabilities through a hub-and-spoke architecture.

The subject speaks to 1ain. 1ain handles — or routes to specialist agents.
One interface. All capabilities.

## The Soul Template
1ain's identity is defined by a soul template — a structured profile built by
profiling the subject across multiple LLM conversations. The soul template is:
- Model-agnostic (works with any LLM)
- Persistent (loaded on every session)
- Self-improving (updated as new signals are observed)
- Owner-controlled (stored on subject's own infrastructure)

## Phase Model
| Phase | Description |
|-------|-------------|
| 1 | 1ain core — personal agent, Telegram interface |
| 1b | Tool integrations — Gmail, Calendar, Drive via MCP |
| 2 | Hermes — task ops specialist agent |
| 3+ | Multi-agent OS — specialist agents for every domain |

## Principles
- Sovereignty: self-hosted, owner-controlled
- Cascade: identity flows from generic to specific, never upward
- Reusability: every component designed for reuse
- Production-grade: security and verification by default
